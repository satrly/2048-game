# 2048-game
2048
Задача
Необходимо создать веб-версию игры 2048, где пользователь должен иметь возможность:

Взаимодействовать с игровым полем
Записать свой рекорд в таблицу лидеров в случае окончания игры
Просмотреть список лидеров
Продолжить игровую сессию с последнего действия
Вернуться на один шаг назад
Начать игру заново
Игра завершается при невозможности совершить движение плиток в любую из 4х сторон

Технические требования
Запрещается использовать CSS-фреймворки (Bootstrap, Foundation и т.д.) и JS-фреймворков (Angular, React, Vue и т.д.).
Разрешается использование препроцессоров CSS (SCSS) по желанию

Этапы разработки
Макет и верстка
Создать базовую HTML-страницу игры: контейнер для тайлов, заголовок, блок с текущим количеством очков, модальное окно/отдельная страница с лидербордом
Создать в контейнере игрового поля/модальным окном контейнер, который будет показываться при окончании игры
При звершении игры должна отображаться следующая информация - информационное сообщение, инпут для ввода имени, кнопка "сохранить результат", кнопка "начать заново" (можно иконкой "обновления"), при вводе имени и нажатии кнопки "сохранить результат" инпут должен скрыться, сообщение измениться на "ваш рекорд сохранен".
Таблица рекордов должна состоять из следующих полей: Имя пользователя, количество очков, дата сабмита рекорда.
Адативный дизайн и медиазапросы. Необходимо обеспечить корректное отображение сайта на разных устройствах: ноутбуках, планшетах и смартфонах.
На десктопе управление должно осуществляться стрелочками на клавиатуре
На мобильных версиях должны отрисовываться 4 кнопки управления (вверх, вправо, вниз, влево). Элементы управления должны отображаться только во время игры (На время просмотра рекордов, сабмита рекорда элементы управления должны быть скрыты)
На усмотрение студента, на мобильных устройствах можно реализовать управление свайпами по полю, в таком случае не нужно реализовывать виртуальные кнопки
Добавление логики
Реализация движений вверх, влево, вправо, вниз
[4, 4, 4, 4] -> [16, 0, 0, 0] (движение влево) (+32 очка - +8, +8, +16)
[16, 16, 2, 2] -> [32, 4, 0, 0] (движение влево) (+36 очков - +32, + 4)
Реализация проверки окончания игры (нет возможных движений)
Появление 1-2 новых плиток с значениями 2/4 на случайных пустых клетках
Отображение текущего счета пользователя
Реализация отмены хода (в случае завершения игры отменить ход нельзя)
Сохранение состояния игры, сохранения лидерборда
Критерии оценивания
Логика игры
1 балл - Сетка 4x4 генерируется динамически с помощью JavaScript
1 балл - После каждого хода корректно появляется 1-2 новых плитки с значением 2 или 4.
1 балл - Слияние плиток однаковых чисел работает во всех направлениях
1 балл - Новая плитка появляется в случайной свободной ячейке (Свободная клетка - клетка, в которой нет тайла на момент завершения предыдущего действия). В начале игры на поле есть 1-3 случайных тайла с значением 2 либо 4
1 балл - Корректный расчет и отображение очков (Очки считаются путем суммирования значений объединенных ячеек за предыдущий ход, пример ниже)
Очки до - 0, состояние ряда - [4, 4, 0, 0], очки после (движение влево) - 8, состояние ряда - [8, 0, 0, 0]
Очки до - 20, состояние ряда - [2, 2, 2, 2], очки после (движение влево) - 36, состояние ряда - [8, 0, 0, 0]
Взаимодействие вне игрового поля
2 балла - Реализована проверка на окончание игры (Нет возможных ходов)
1 балл - Кнопка "Начать заново" сбрасывает состояние поля и счет (Без перезагрузки страницы)
1 балл - Реализована кнопка "Отмена хода" (undo)
1 балл - В таблице отражается топ-10 рекордов, которые сохранил пользователь. (Хранение в localStorate)
Взаимодействие с игрой
1 балл - Поддержка управления кнопками клавиатуры для десктопа и свайпами/виртуальными кнопками на мобильных устройствах
1 балл - Состояние игры сохраняется в localStorage
Визуальное отображение
1 балл - Каждое значение плитки имеет свой цвет
2 балла - Плавная анимация движения и слияния плиток
Бонус
Дизайн и расположение элементов полностью остаётся на усмотрение студента. Дополнительно, три работы, которые преподаватели сочтут наиболее интересными, получат бонусные баллы.

Штрафы
Использование запрещённых фреймворков - работа оценивается в 0 баллов. Работа должна быть реализована с использованием только чистого HTML, CSS (препроцессоры ок) и JavaScript.
Сдача работы после дедлайна без уважительной причины - 50% от полученных баллов.
